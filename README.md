# DisasmProof

Мини-программа для снятия машинного кода, который JIT генерирует для `foreach` по `IEnumerable<int>`. Написана для статьи **«Бенчмаркая foreach и энумераторы: аллокации, которые прятались 10 лет»** — здесь лежит пруф к разделу «Смотрим машинный код».

## Зачем

Один и тот же метод:

```csharp
static int SumOverInterface(IEnumerable<int> values)
{
    int sum = 0;
    foreach (int i in values)
    {
        sum += i;
    }

    return sum;
}
```

на .NET 8 и .NET 10 компилируется в принципиально разный машинный код:

| | .NET 8 | .NET 10 |
|---|---|---|
| Энумератор | создается в куче (32 B на вызов) | не создается вообще |
| Проверка типа | на каждой итерации | один раз на входе в метод |
| Цикл | виртуальные вызовы на запасном пути | прямой индексный проход по массиву |
| Размер кода | 460 байт | 193 байта |

JIT в .NET 10 распознает, что за интерфейсом прячется `int[]`, и переписывает `foreach` в обычный `for` — энумератора нет ни в куче, ни на стеке.

## Как снять листинги

Нужны SDK .NET 8 и .NET 10. Запусти `snap.bat` — или руками:

```cmd
dotnet build -c Release

set DOTNET_JitDisasm=*SumOverInterface*

bin\Release\net8.0\DisasmProof.exe  > disasm_net8.txt  2>&1
bin\Release\net10.0\DisasmProof.exe > disasm_net10.txt 2>&1

set DOTNET_JitDisasm=
```

В каждом файле метод встретится несколько раз — JIT перекомпилирует его по мере прогрева (Tier0 → Tier0 с PGO-инструментацией → Tier1). Смотреть нужно последнюю версию, **Tier1** — это финальный оптимизированный код.

## Файлы

- `Program.cs` — сама программа: метод гоняется 200 000 раз, чтобы дойти до Tier1
- `snap.bat` — снимает оба листинга одной командой
- `disasm_net8.txt`, `disasm_net10.txt` — готовые листинги с моей машины (i9-10900KF, .NET 8.0.28 / 10.0.9)

## Грабли, на которые я наступил

Не объявляйте измеряемый метод локальной функцией внутри top-level кода — компилятор переименует его во что-то вродe `<<Main>$>g__SumOverInterface|0_0`, и фильтр `DOTNET_JitDisasm` по человеческому имени его не найдет. Метод должен лежать в обычном классе, а фильтр надежнее со звездочками: `*SumOverInterface*`.
